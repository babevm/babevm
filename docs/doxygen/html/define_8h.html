◆ BVM_DEBUG_HEAP_GC_ON_ALLOC

#define BVM_DEBUG_HEAP_GC_ON_ALLOC   0

When set, the VM performs a GC before every memory allocation.

Default is disabled.

Note
This is pretty severe stuff - it causes a complete garbage collection cycle before every memory allocation - thus if there are to be memory issues or GC issues, this will help find them.

Enables debugger support for the 'Signature' class file attributes for fields, methods, and classes.

This adds a utfstring pointer to every field, method and class and also creates a utfstring for each signature encountered. So be wary, it may consume quite some heap. ◆ BVM_DIRECT_THREADING_ENABLE

#define BVM_DIRECT_THREADING_ENABLE   0

"Direct Threading" takes advantage of the GCC ability to treat labels as first class values and are used with "goto" to speed up a 'switch'-like statement. If GNUC is defined we'll define BVM_DIRECT_THREADING_ENABLE=1 so that the interp dispatch can be sped up. Probably by 5% to 10%.

On GNUC, direct threading will be enabled by default, but can be turned off by defining BVM_DIRECT_THREADING_ENABLE=0.

Note
The use of direct threading avoids a few extra machine instructions that a 'switch' statement generates to range check. If GCC, unless you're having issues, direct threading is recommended.
