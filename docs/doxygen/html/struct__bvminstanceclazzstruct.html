◆ constant_pool

BVM_COMMON_CLAZZ_INFO bvm_clazzconstant_t* constant_pool

include all common class stuff The constant pool associated with the class. The first constant is the number of constants and all subsequent constants are constants as defined in the class file. Why? The class file spec has constants as 1-based, not zero-based. Using a structure like this means direct access to a constant from the index defined in bytecode. We gain 4 bytes per class, but the speed of access is worth it - we access the constant pool a lot.

◆ instance_fields_count

bvm_uint16_t instance_fields_count

Holds the cumulative number of non-static fields (including fields in superclasses) for this class def. Using this count, we can easily calculate how much memory an instance of this class will consume. memory = instance_fields_count * sizeof(bvm_cell_t). Easy.
