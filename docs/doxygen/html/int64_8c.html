Detailed Description

Functions for bvm_int64_t support.

Author
Greg McCreath
Since
0.0.10

Software implementation of Java 'long' arithmetic. Uses the "Arbitrary Precision" (XP) code from "C Interfaces and Implementations: Techniques for Creating Reusable Software". I'm sure all this can be done a lot more efficiently and with much less hassle and code.

But for now this is it.

The XP code is all little-endian and only works on positive numbers. Hence there is a bit of gymnastics in here for negatives and positives and endianness.
