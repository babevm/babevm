Detailed Description

Debugger transport interface. The Babe VM uses a populated instance of this struct to call transport functions for communicating with a debugger.

◆ accept

bvmd_transport_error_t(* accept(bvm_int32_t accept_timeout, bvm_int32_t handshake_timeout)

Accepts in incoming debugger connection.

Parameters
accept_timeoutsuggested milliseconds to wait for an incoming connection. To never timeout, specify a timeout of BVMD_TIMEOUT_FOREVER.
handshake_timeoutsuggested timeout milliseconds for debugger handshake.
Returns
BVMD_TRANSPORT_ERROR_ILLEGAL_STATE if a debugger connection is already open, or the VM is not listening for connections, BVMD_TRANSPORT_ERROR_TIMEOUT if an accept did not happen within the timeout period, BVMD_TRANSPORT_ERROR_IO_ERROR for everything else, or BVMD_TRANSPORT_ERROR_NONE if all goes well.

◆ attach

bvmd_transport_error_t(* attach(const char *address, bvm_int32_t attach_timeout, bvm_int32_t handshake_timeout)

Function pointer for attaching to a remote debugger at a given address. The meaning of the address is transport dependent.

Parameters
addressthe address to connected to. Max 128 chars.
attach_timeoutsuggested timeout for attaching to the debugger. Implementations may use this suggestion and follow it is possible, or do what they can.
handshake_timeoutsuggested timeout for the handshake to the debugger. Implementations may use this suggestion and follow it is possible, or do what they can.
Returns
BVMD_TRANSPORT_ERROR_ILLEGAL_ARGUMENT the format of the address is invalid, BVMD_TRANSPORT_ERROR_ILLEGAL_STATE if a connection is already open, BVMD_TRANSPORT_ERROR_IO_ERROR if any IO error occurs, or BVMD_TRANSPORT_ERROR_NONE if all is good.
